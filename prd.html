Product Requirements Document
Shopify Foresight
A native proactive intelligence layer for Shopify Admin — surfacing prioritized insights, anomaly alerts, and one-click actions for every merchant on the platform.

01 Overview

Problem statement

Shopify's native analytics dashboard is reactive by design. It shows merchants what already happened — sales yesterday, sessions last week, top products last month. It doesn't yet surface emerging risks, flag anomalies, or suggest actions before problems show up in revenue.

Most merchants aren't data analysts. They're founders and operators juggling everything at once, and the question they most often have at the start of a day is simply: what should I pay attention to today? That answer takes real digging to find in the current admin — and for many merchants, it never gets found at all.

Shopify already holds a remarkable amount of useful data — order history, real-time sessions, payment flows, inventory signals, and patterns across millions of stores. Foresight is an idea for making that data work a little harder for merchants, proactively rather than on demand.

The opportunity: Merchants who get timely, relevant signals from their platform are better positioned to act on growth opportunities and catch problems early. Foresight is designed around that idea.

Goals

GoalRationale
Surface 3–5 prioritized, actionable insights daily for every merchantReplace passive pull with active push — make the admin feel like a morning briefing
Detect anomalies (conversion drops, inventory risk, revenue spikes) in real timeMost merchants don't monitor dashboards continuously — anomalies go unnoticed for days
Enable natural language Q&A on store dataLower the skill barrier — non-technical founders should get answers without SQL or analyst support
Drive daily-active-admin behavior across all plan tiersMerchants who open admin daily are more engaged, more likely to upgrade, and less likely to churn
Use cross-merchant benchmark data as a native moatNo third-party app can provide category-level comps — this is Shopify's unique structural advantage

Non-goals (v1)

The following are explicitly out of scope for the initial launch and should not be used to evaluate Phase 1 success:

Non-goalReasoning
Ad channel attribution (Meta, Google, TikTok pixel)Triple Whale owns this space; Foresight does not need a first-party pixel to be valuable in v1
Financial integrations (Xero, QuickBooks)Cross-system P&L is a Luca AI differentiator — adds complexity without serving the core use case
Replacing Shopify Analytics dashboardForesight sits alongside existing reports as an intelligence layer, not a replacement
Custom report builderShopifyQL already serves power users — Foresight targets the non-technical majority
Mobile app (Shopify mobile) integrationPhase 2 consideration — admin web is the primary surface for launch

03 User stories

  • Founder-Operator
    As a merchant, I want to open Shopify Admin and immediately see what needs my attention today, so that I don't have to manually dig through reports to find problems.
    Acceptance criteria
    • Foresight feed is visible on Admin home or as a dedicated nav item without any setup
    • At least 1 insight is surfaced within the first 24 hours of store activity
    • Each insight has a plain-English title, a 1–2 sentence explanation, and a severity indicator
  • Growth Hire
    As a growth manager, I want to be alerted when conversion rate drops significantly on a specific device type, so that I can investigate and fix the issue before it compounds over days.
    Acceptance criteria
    • Conversion anomaly is detected within 4 hours of the drop occurring
    • Alert specifies device type, magnitude of drop, and comparison baseline
    • Notification delivered via Admin notification center and optionally via email
  • Founder-Operator
    As a merchant, I want to ask plain English questions about my store and get answers without needing SQL or external tools, so that I can make data-driven decisions without hiring an analyst.
    Acceptance criteria
    • Q&A accepts free-text questions about orders, products, sessions, and customer behavior
    • Responses synthesize across multiple data dimensions in a single answer
    • Response time is under 5 seconds for standard queries
    • System gracefully handles out-of-scope questions with a helpful fallback
  • Scaling Brand
    As a head of e-commerce, I want to see how my store's key metrics compare to similar brands in my category, so that I can understand whether a metric issue is specific to my store or an industry-wide trend.
    Acceptance criteria
    • Benchmark data is anonymized and aggregated — no individual store data exposed
    • Category matching uses product taxonomy and GMV band, not self-reported data
    • Benchmarks shown for conversion rate, return rate, repeat purchase rate, and AOV
    • Benchmark data is labeled with sample size and methodology footnote
  • Founder-Operator
    As a merchant, I want to take action on an insight without leaving the Foresight screen, so that the time between seeing a problem and fixing it is as short as possible.
    Acceptance criteria
    • Each insight surfaces 1 recommended action with a clear CTA
    • Actions that can be executed natively (inventory reorder, discount creation) happen in-context
    • Actions that require deeper configuration link directly to the relevant Admin page

06 Phased roadmap

Phase 1 · Q3 2026
Anomaly alerts
  • Conversion rate drop detection
  • Checkout abandonment spike
  • Inventory risk (days-of-stock)
  • Revenue outlier (up & down)
  • Admin notification center delivery
  • Email opt-in for critical alerts
  • Insight dismiss / snooze
  • Available on all plan tiers
Phase 2 · Q4 2026
Insight feed + Q&A
  • Daily prioritized insight feed (3–5 items)
  • Trending product detection
  • Customer cohort shifts
  • Natural language Q&A on store data
  • One-click actions (inventory, discounts)
  • Personalization based on feedback
  • Shopify / Advanced plan gating for Q&A
Phase 3 · Q1 2027
Network benchmarks
  • Anonymous cross-merchant benchmarks
  • Category + GMV band segmentation
  • Conversion rate, return rate, AOV, LTV
  • Seasonal benchmark adjustments
  • "Your store vs. similar stores" view
  • Plus plan gating for full benchmark suite
Phase 3 is the strategic moat. Cross-merchant benchmarks are the only capability no third-party app can replicate. Delay is acceptable if it means doing it right — minimum 50-store cohort floor, legal sign-off on anonymization, and opt-in from merchants in beta cohort.

07 Risks & mitigations

High
Alert fatigue if early insights aren't relevant enough
If the first wave of insights feels generic or too frequent, merchants will ignore or dismiss the feature before it has a chance to demonstrate value. Getting the signal quality right early matters a lot.
Mitigation: Start with a small set of high-confidence anomaly types only. Set conservative thresholds (e.g., conversion drops >15% vs. baseline before alerting). Use dismissal rate as a primary signal quality metric, and run a closed beta with a small merchant cohort before broad rollout.
High
Relationship with analytics app partners
Shopify has a large and valuable app ecosystem. A native intelligence feature that overlaps with existing analytics apps could create tension with partners. Positioning and communication matters here.
Mitigation: Frame Foresight as a foundation for merchants who haven't adopted dedicated analytics tools yet — not a replacement for tools like Triple Whale. Explore a data API that lets partner apps surface their data within Foresight rather than compete with it. Communicate with key partners before any public launch.
Medium
Insight accuracy — building merchant trust takes time
A single misfired "urgent" alert — say, flagging a conversion drop caused by a data lag rather than a real issue — can erode trust quickly. Merchants need to be able to rely on what Foresight surfaces.
Mitigation: Require a 95% confidence threshold before surfacing anomaly alerts. Include a brief "how we calculated this" note on each insight. Add a simple feedback mechanism ("Was this useful?") from day one to help the model improve over time.
Medium
Benchmark data requires careful privacy and legal review
Cross-merchant benchmarks are exciting but need to be handled thoughtfully. Cohort sizes that are too small could inadvertently surface competitive information, and regional privacy regulations (GDPR, CCPA) add complexity.
Mitigation: Phase 3 requires full legal review before any data pipeline work begins. Enforce a minimum cohort size of 50 stores per benchmark cell. Consider an opt-in model for merchants contributing to the benchmark pool.
Low
Q&A responses that miss the mark
Natural language Q&A on store data is powerful but requires guardrails. Responses that are plausible but inaccurate could lead merchants to make decisions based on bad information.
Mitigation: Always ground responses in cited data (e.g., "Based on your last 847 orders…"). Include a confidence signal. For out-of-scope or ambiguous questions, return a helpful fallback that links to the relevant Analytics report rather than attempting a guess.
